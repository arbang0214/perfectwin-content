## 6. 인사이트

### 1. 🚨 급격한 트래픽 폭증 (전일 대비 43배) - 허수 트래픽 의심

**현상**: 세션이 16건에서 694건으로 4,237% 급증했으나, 참여율은 37.5%에서 18.73%로 50% 하락. 세션당 페이지뷰도 1.38에서 1.05로 감소.

**왜 이런 일이 생겼나**: Direct 트래픽이 679건으로 전체의 97.8%를 차지하며, 이 중 대부분이 참여하지 않은 세션. 평균 체류시간은 4분 42초로 길지만 참여율과 페이지뷰가 낮아 "방치 탭" 패턴. 특히 미국에서 466세션이 유입됐지만 참여율은 12.66%에 불과.

**비즈니스 임팩트**: 허수 트래픽으로 실제 잠재고객 유입이 아님. 서버 리소스 소모와 데이터 노이즈만 증가. 실제 전환으로 이어질 가능성 낮음.

**→ 액션**: GA4에서 봇 필터링 설정 확인. 미국 IP 대역에서 참여율 10% 미만 세션을 봇으로 분류하여 필터링 적용. 내일 데이터에서 동일 패턴 반복 시 서버 액세스 로그와 교차 확인.

### 2. 검색 유입 완전 차단 상태 - SEO 가시성 제로

**현상**: Organic Search는 7세션(1.0%)에 불과하고 모두 참여율 0%. Bing에서만 7건 유입되었으나 체류시간 0초로 즉시 이탈.

**왜 이런 일이 생겼나**: Google GSC 데이터가 null이고 Bing도 노출/클릭 0건. 검색엔진에서 사이트가 아예 발견되지 않거나 인덱싱되지 않은 상태. 기존 7일 평균에서도 검색 트래픽이 거의 없었던 점으로 보아 지속적인 문제.

**비즈니스 임팩트**: SAP 테스트 자동화를 검색하는 잠재고객이 PerfecTwin을 발견할 수 없음. B2B SaaS에서 검색 유입은 고품질 리드의 주요 소스인데 완전히 차단된 상태.

**→ 액션**: Google Search Console에서 사이트맵 재제출. robots.txt 파일 확인하여 크롤링 차단 여부 점검. 메인 페이지와 주요 제품 페이지의 title/meta description 최적화. "SAP testing automation", "SAP test automation platform" 등 타겟 키워드 포함하여 페이지 재작성.

### 3. 전환 페이지 접근성 문제 - 데모 요청 동선 개선 필요

**현상**: 데모 요청 페이지(/contact-us/request-demo)는 16 pageViews, 23 users로 집계 불일치. 랜딩페이지로 직접 진입한 9세션 중 66.67%가 이탈.

**왜 이런 일이 생겼나**: pageViews < users는 GA4 집계 오류 신호. 실제로는 데모 페이지 도달률이 매우 낮음. 메인 페이지에서 685 pageViews 중 16건만 데모 페이지로 이동(2.3%)하여 전환 동선이 매우 약함.

**비즈니스 임팩트**: 트래픽은 있지만 전환으로 이어지지 않음. 694세션 중 실제 데모 요청으로 연결된 비율이 극히 낮아 매출 기회 손실.

**→ 액션**: 메인 페이지에 "Request Demo" CTA 버튼을 above-the-fold 영역에 더 눈에 띄게 배치. 현재 데모 페이지의 이탈률 66.67%를 줄이기 위해 폼 필드 수 최소화(이메일, 회사명, 연락처만). 데모 요청 완료 이벤트 추적 설정하여 실제 전환율 측정.

### 4. 모바일 접근성 극히 제한적 - B2B도 모바일 최적화 필요

**현상**: 모바일 세션은 5건(0.4%)에 불과하지만 참여율은 60%로 데스크톱(18.67%)보다 3배 높음. 모바일 체류시간은 2분 30초로 데스크톱보다 짧지만 참여도는 높음.

**왜 이런 일이 생겼나**: B2B SaaS 특성상 업무 시간 데스크톱 접근이 주류이지만, 모바일 방문자는 높은 참여율을 보여 실제 관심도가 높음. 모바일 접근성이 제한적이어서 잠재적 리드를 놓치고 있을 가능성.

**비즈니스 임팩트**: 모바일에서 사이트를 발견한 잠재고객이 적절한 경험을 받지 못해 기회 손실. 특히 출장 중이거나 회의실에서 모바일로 제품을 검토하는 의사결정자들이 접근하기 어려움.

**→ 액션**: 메인 페이지와 데모 요청 페이지의 모바일 반응형 디자인 점검. 모바일에서 CTA 버튼 크기와 위치 최적화. Google PageSpeed Insights로 모바일 성능 측정하여 로딩 속도 개선. 모바일 사용성 테스트 진행.
