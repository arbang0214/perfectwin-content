### 1. **Bing 검색 트래픽이 모두 봇으로 추정됨**
- **현상**: Bing 검색 3세션이 모두 참여율 0%, 체류시간 0초, users = 0으로 기록됨. 전일(0세션) 대비 +3세션이지만 실질적 방문자는 없음.
- **왜 이런 일이 생겼나**: 동일한 패턴(참여율 0%, 체류시간 0초, users 0)은 전형적인 봇 트래픽 신호. Bing의 크롤러가 사이트를 스캔하는 과정으로 보임.
- **비즈니스 임팩트**: 실제 잠재고객 유입이 아니므로 데모 요청으로 이어질 가능성 0%. 트래픽 수치만 부풀리는 노이즈.
- **→ 액션**: GA4 필터에서 Bing 봇 트래픽을 제외하도록 설정. 또는 분석 시 users = 0인 세션은 봇으로 간주하고 제외.

### 2. **직접 유입의 품질이 검색 유입보다 훨씬 우수함**
- **현상**: Direct 채널 참여율 50% vs Organic Search 20%. 직접 유입 평균 체류시간 3분 20초 vs Google 검색 5분 22초는 높지만, Bing 봇을 제외하면 Google만 2세션에 참여율 50%.
- **왜 이런 일이 생겼나**: 직접 유입(북마크, URL 입력)은 의도성이 높은 방문자. 반면 검색 유입은 Bing 봇 3세션이 평균을 끌어내림. Google 검색 2세션은 실제로는 50% 참여율로 직접 유입과 동등.
- **비즈니스 임팩트**: 의도성 높은 방문자일수록 데모 요청 전환 가능성 높음. 현재 직접 유입이 더 따뜻한 리드.
- **→액션**: 직접 유입 증대를 위해 이메일 시그니처, 명함, LinkedIn 프로필에 perfectwin.ai URL 적극 노출. 뉴스레터나 이메일 아웃리치에서도 직접 링크 사용.

### 3. **어제 대비 체류시간이 극적으로 개선되었지만 여전히 낮은 탐색성**
- **현상**: 평균 체류시간이 0분 1초 → 2분 25초로 144초 증가. 하지만 세션당 페이지뷰는 1.0 → 0.9로 오히려 감소.
- **왜 이런 일이 생겼나**: 어제는 100% 이탈률의 봇성 트래픽이었고, 오늘은 실제 방문자가 늘어남. 하지만 긴 체류시간에도 불구하고 페이지 탐색이 적다는 것은 메인 페이지에서 읽고 있지만 다른 페이지로의 유도가 약함을 의미.
- **비즈니스 임팩트**: 관심은 있지만(긴 체류) 행동으로 이어지지 않음(낮은 탐색). 잠재고객이 와서 보고 있지만 다음 단계로 넘어가지 못하는 상황.
- **→ 액션**: 메인 페이지의 CTA 버튼 위치와 문구 개선. "데모 요청" 버튼을 hero section과 중간, 하단에 3번 배치. "왜 PerfecTwin인가?" 섹션 하단에 "/why-perfectwin" 내부 링크 추가.

- **→ 액션**: SAP 관련 레퍼럴 트래픽을 위한 별도 랜딩 페이지 제작 검토. 또는 현재 메인 페이지에서 "SAP 테스트 자동화" 메시지를 더 명확하게 전면에 배치.
