# PerfecTwin Brand Guide

## Product Identity

- **제품명**: PerfecTwin (항상 CamelCase, "Perfectwin" 또는 "perfectwin" 금지)
- **카테고리**: SAP 테스트 자동화 도구
- **타겟 고객**: SAP Basis 관리자, ABAP 개발자, QA 리드, IT 디렉터

## Core Differentiators

| 차별화 포인트 | 핵심 메시지 |
|---|---|
| 50x 실행 속도 | 기존 도구 대비 50배 빠른 테스트 실행 |
| Data Extractor | SAP 테스트 데이터를 자동 추출·관리 |
| No-code 드래그앤드롭 | 비개발자도 테스트 시나리오 구성 가능 |
| Pre-built 템플릿 | SAP 표준 프로세스 기반 즉시 사용 가능한 테스트 |
| 가격 경쟁력 | 엔터프라이즈 기능을 합리적 가격에 제공 |

## Tone by Channel

### 블로그 (blog-ko, blog-en)
- **톤**: 교육적, 가치 중심
- 독자에게 실질적 인사이트를 먼저 제공
- PerfecTwin 직접 언급은 **마지막 CTA 섹션에만** 허용
- 나머지 본문은 방법론·프레임워크 중심으로 서술

### LinkedIn 회사 (linkedin-company)
- **톤**: 권위 있고 공식적
- 업계 리더로서의 포지셔닝
- 데이터, 리서치, 모범 사례 중심
- 이모지 사용 절제 (섹션 구분용 1~2개만)

### LinkedIn 개인 — ARUM (linkedin-personal)
- **톤**: 1인칭, PM 인사이트 공유
- "저는~", "제가 경험한~" 시점
- 제품 홍보가 아닌 개인 전문성 기반 스토리텔링
- 실패담, 배운 점, 현장 이야기 중심

### X (Twitter)
- **톤**: B급 감성 — 위트, 밈 느낌, 가볍지만 핵심은 날카롭게
- SAP 실무자가 공감하며 리트윗할 수 있는 수준
- 업계 클리셰 비틀기, 밈 포맷 적극 활용
- 이모지 1~2개까지

## Visual Identity

- **Primary Palette**: Dark Navy + Electric Blue
- 블로그 대표 이미지, 소셜 카드 모두 이 기조 유지
- Ideogram 이미지 생성 시 컬러 가이드로 활용

## Messaging Do's & Don'ts

### Do
- 카테고리 교육 접근 ("SAP 테스트 자동화를 왜 해야 하는가")
- 독자의 pain point에서 출발
- 구체적 수치·사례 활용

### Don't
- ❌ SAP ALM 코인스톨레이션(co-installation) 관련 내용은 외부 메시징에 포함 금지
- ❌ Tricentis Tosca 등 경쟁사 직접 비교/공격 금지 → 카테고리 교육으로 우회
- ❌ 과장된 마케팅 어조 ("혁신적", "게임체인저" 등 남용 금지)
- ❌ 기술적으로 검증되지 않은 수치 사용 금지
